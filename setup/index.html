    How to log in

    The Home Page

    Step Details Additional Information
    1 If no Teamspace is available follow the steps to proceed with setting up a Teamspace and Loop
    2 If a Teamspace is available you will be directed to your homepage and it will provide you an overview your Loops
    2.1 If no Loop exists, proceed with Setup a Loop
    2.2 If your loop is empty proceed with Setting up Demo Data
    2.3 If your loop has data proceed with the Demo Flow or Self Enablement

    Setup a Teamspace and Loop

    Setup a Teamspace

    Step Details Additional Information
    1 Click on the Teamspace icon to view your Teamspaces or create a new one
    2 Click on Create TeamSpace
    3 Enter a name and a Description
    Example Name: 'Demo'
    Example Description: 'Teamspace for Demos'
    4 press Next Button to proceed
    5 You can search and add Teammembers to a Teamspace
    6 Teammember is added to a list. Proceed to add more members if needed.
    7 Press Create to proceed
    8 Teamspace creation will take a few moments, as soon as it finishes a success dialog will be shown
    9 Teamspace created. Click on Go to Teamspace for next steps
    10 When the creation of a TeamSpace is finished you will be forwarded
    to your empty loops home page for your new TeamSpace
    Please proceed with Setting up a Loop

    Setup a Loop

    Step Details Additional Information
    1 Press the Create Loop Button
    2 In the Loop creation Dialog enter a name and a description
    3 press the Next Button to proceed
    4 You can search and add a Teammember to the Loop
    5 a new Member is added
    6 Press the Create Button to start Loop creation
    7 Loop creation can take a few moments, as soon as it finishes a Success screen will be shown
    8 Creation is successfull
    9 Press now Go to my Loops Button to proceed
    10 As your Loop has no data please proceed with Setting up Demo Data

    Setup Demo Data

    Importing Demo Data into Plan

    Step Details Additional Information
    1 Switch to Plan
    1.1 by either pressing "Let's go to Plan" button on its tile
    1.2 by using the central app switcher on the top left of your screen
    1.3 by using the button/link from What would you like to do today section
    2 Click on Let's Go on the "EchoL" application
    2.1 if you get a Welcome Screen click to close to proceed with data import
    3 click in the Sidebar
    3.1 on Configuration
    3.2 Use the Import Feature of plan to load sample data
    4 NOTE Download the Demo Data File onto your local drive
    4.1 Click oon EchoLogic Zip File from files section
    4.2 and save the file on your local drive (best into Downloads)
    5 In the Dialog for importing record data
    5.1 click the Choose File Button
    5.2 select the sample data file (from Step 2) to be imported
    5.3 Now we have the file selected
    5.4 Press the Import Button to start
    6 The Import process starts.
    6.1 By clicking on Details the process output can be viewed
    6.2 Import has finished
    6.3 Press the close button (bottom right on the screen)
    7 verify that data has been loaded
    7.1 Click on Project Board icon on the Sidebar
    7.2 Your board should look similar to this screenshot

    Importing Demo Data into Control

    Task: Copy files and folders from Demo App repository URL: https://github.com/DevOpsAutomationLabs/EchoLogic_DemoApp.git into target repository.

    Step Details Additional Information
    1 Switch to Control
    1.1 by either pressing "Let's go to Control" button on its tile
    1.2 by using the central app switcher on the top left of your screen
    1.3 by using the button/link from What would you like to do today section
    Setup a Personal Access Token
    Step Details Additional Information
    1 Click on the top right on Profile and Settings
    1.1 Select Settings
    2 Click on Access Tokens in the menu
    2.1 click on the Select Permissions section to open settings
    2.2 Set "issue" to "Read/Write"
    2.3 Set "repository" to "Read/Write"
    2.4 Set "user" to "Read"
    2.5 Click on Generate Token Button
    3 The new Token is created and shown
    NOTE:
    3.1 Copy and Paste the token into a text file for reference!
    Clone Target Repository
    Step Details Additional Information
    1 Open a Terminal to clone the target and source repositories
    2 cd into the Documents folder cd Documents
    3 Clone the target repository
    3.1 Git Clone command git clone https://devops.automation.techzone.ibm.com/control/Demo/EchoLogic.git"
    3.2 Provide Username: demo-devops.loop
    3.3 Provide access token (copy from textfile)
    4 The target repository is cloned
    4.1 check the folder with ls EchoLogic
    5 If needed Getting back to Documents cd&cd Documents
    6 Clone the source repository
    6.1 Git Clone Command git clone https://github.com/DevOpsAutomationLabs/EchoLogic_DemoApp.git
    6.2 check Documents folder ls
    7 Copy some folders and files from Source Repo to Target Repo
    7.1 Copy docs folder cp -r EchoLogic_DemoApp/docs EchoLogic/
    7.2 Copy files folder cp -r EchoLogic_DemoApp/files EchoLogic/
    7.3 Copy src folder cp -r EchoLogic_DemoApp/src EchoLogic/
    7.4 Copy the Readme file cp -r EchoLogic_DemoApp/README.md EchoLogic/
    8 Check target repo folder ls EchoLogic
    9 Push changes
    9.1 Add changes git add .
    9.2 Commit Message git commit -m "upload"
    9.3 Push changes git push
    9.4 Provide username and token (same as with clone)
    9.4.1 demo-devops.loop
    9.4.2 Provide access token (copy from textfile)
    9.5 push finishes
    10 Switch back to Control and view Repo
